01

Component Structure

Angular components are class-based with decorators. React components are plain functions that return JSX — no decorators, no separate files, no boilerplate.

Angular
// app.component.ts
import { Component, OnInit } from '@angular/core';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent implements OnInit {
  title = 'my-app';

  constructor(private myService: MyService) {}

  ngOnInit(): void {
    // initialization logic
  }
}
React + Hooks
// App.tsx
import { useEffect } from 'react';
import { useMyService } from './hooks/useMyService';

export default function App() {
  const title = 'my-app';
  const myService = useMyService();

  useEffect(() => {
    // initialization logic
  }, []);

  return (
    <div className="p-4">
      <h1 className="text-2xl font-bold">{title}</h1>
    </div>
  );
}
  • No @Component decorator — just export a function
  • No selector — components are imported and used as JSX tags directly
  • No separate .html / .css files — JSX and Tailwind classes live together
  • constructor() → function body or useState / custom hooks
  • ngOnInit() → useEffect(() => {}, [])

for="..."htmlFor="..."

[style.color]="val"style={{ color: val }}

Data Binding

Angular has 4 binding types. React uses one-way data flow — two-way binding is achieved with controlled components (value + onChange pair).

Angular — ngModel two-way
// Two-way binding
name = '';
// template:
<input [(ngModel)]="name" />
<p>Hello {{ name }}</p>
React — controlled component
const [name, setName] = useState('');

<input
  value={name}
  onChange={e => setName(e.target.value)}
  className="border p-2 rounded"
/>
<p>Hello {name}</p>

Lifecycle Hooks

Angular has 8 lifecycle hooks. React consolidates them into useEffect with different dependency arrays.

constructor()
↓
useState() init / fn body
Runs once during component creation
ngOnInit()
↓
useEffect(() => {}, [])
After first render, empty deps array
ngOnChanges()
↓
useEffect(() => {}, [dep])
Runs when specified dependency changes
ngOnDestroy()
↓
return () => {} (cleanup fn)
Cleanup inside useEffect return
ngAfterViewInit()
↓
useEffect + useRef
After DOM is painted and accessible
ngAfterViewChecked()
↓
useEffect (no deps)
After every render cycle completes
Angular
export class MyComponent
  implements OnInit, OnDestroy, OnChanges {

  ngOnInit() { /* setup */ }

  ngOnChanges(changes: SimpleChanges) {
    /* prop changed */
  }

  ngOnDestroy() { /* cleanup */ }
}
React
useEffect(() => {
  console.log('mounted'); // ngOnInit
  return () => {
    console.log('unmounted'); // ngOnDestroy
  };
}, []); // empty = run once

useEffect(() => {
  // ngOnChanges — runs when someProp changes
  console.log('someProp changed:', someProp);
}, [someProp]);

Input & Output (Props & Events)

@Input() → props passed down. @Output() + EventEmitter → callback function as a prop (convention: onXxx).

Angular
export class ChildComponent {
  @Input() title: string = '';
  @Input() count: number = 0;
  @Output() clicked =
    new EventEmitter<string>();

  handleClick() {
    this.clicked.emit('Hello from child');
  }
}

<!-- Parent -->
<app-child
  [title]="parentTitle"
  [count]="5"
  (clicked)="onChildClick($event)"
></app-child>
React
interface ChildProps {
  title: string;
  count: number;
  onClicked: (value: string) => void;
}

function Child({ title, count, onClicked }: ChildProps) {
  return (
    <button onClick={() => onClicked('Hello from child')}>
      {title} - {count}
    </button>
  );
}

// Parent usage:
<Child title={parentTitle} count={5}
       onClicked={handleChildClick} />

Services & Dependency Injection

Angular injects services via constructor. React replaces DI with custom hooks (for stateless logic) and Context API (for singletons/global state).

Angular Service
@Injectable({ providedIn: 'root' })
export class ApiService {
  constructor(private http: HttpClient) {}

  getUsers() {
    return this.http.get<User[]>('/api/users');
  }
}
// Component injection:
constructor(private api: ApiService) {}
this.api.getUsers().subscribe(...);
React Custom Hook
function useApiService() {
  const getUsers = async () => {
    const res = await fetch('/api/users');
    return res.json();
  };
  return { getUsers };
}

// Component usage:
const { getUsers } = useApiService();
useEffect(() => {
  getUsers().then(setUsers);
}, []);
💡
For singleton services (shared state across the app), use the Context API: wrap your app in a <Provider> and consume with useContext() — this replaces Angular's providedIn: 'root'.

Forms

Template-Driven → Controlled Components
Angular ngModel
<form #f="ngForm" (ngSubmit)="onSubmit(f)">
  <input name="email"
    [(ngModel)]="email"
    required email
    #emailField="ngModel" />
  <span *ngIf="emailField.invalid
    && emailField.touched">
    Invalid email
  </span>
  <button [disabled]="f.invalid">Submit</button>
</form>
React Controlled
const [email, setEmail] = useState('');
const [touched, setTouched] = useState(false);
const isValid = /\S+@\S+\.\S+/.test(email);

<form onSubmit={e => { e.preventDefault(); onSubmit(); }}>
  <input value={email}
    onChange={e => setEmail(e.target.value)}
    onBlur={() => setTouched(true)}
    className="border p-2 rounded" />
  {!isValid && touched &&
    <span className="text-red-500">Invalid email</span>}
  <button disabled={!isValid}>Submit</button>
</form>
✅
Reactive Forms → Use react-hook-form for complex validation. It mirrors Angular's Reactive Forms API with register, handleSubmit, and formState.errors.

HTTP Calls

HttpClient + Observable
this.http.get<User[]>('/api/users')
  .subscribe({
    next: users => this.users = users,
    error: err => console.error(err)
  });
React Query (recommended)

  = useQuery({
    queryKey: ['users'],
    queryFn: () => fetch('/api/users')
                    .then(r => r.json()),
  });
⚠️
React Query is the closest equivalent to Angular's HttpClient + async pipe combo — it handles caching, loading states, refetching, and error handling automatically.

Custom Directives

Angular attribute directives attach behavior to DOM elements. React achieves this with custom hooks (for DOM logic) or wrapper components.

Angular Directive
@Directive({ selector: '[appHighlight]' })
export class HighlightDirective {
  @Input() appHighlight = 'yellow';
  constructor(private el: ElementRef) {}

  @HostListener('mouseenter') onEnter() {
    this.el.nativeElement.style
      .backgroundColor = this.appHighlight;
  }
  @HostListener('mouseleave') onLeave() {
    this.el.nativeElement.style
      .backgroundColor = '';
  }
}
React Custom Hook
function useHighlight(color = 'yellow') {
  const ref = useRef<HTMLElement>(null);
  useEffect(() => {
    const el = ref.current;
    if (!el) return;
    const enter = () =>
      el.style.backgroundColor = color;
    const leave = () =>
      el.style.backgroundColor = '';
    el.addEventListener('mouseenter', enter);
    el.addEventListener('mouseleave', leave);
    return () => {
      el.removeEventListener('mouseenter', enter);
      el.removeEventListener('mouseleave', leave);
    };
  }, [color]);
  return ref;
}
// Usage:
const ref = useHighlight('lightblue');
<p ref={ref}>Hover me</p>

Error Handling

Global ErrorHandler
@Injectable()
class GlobalErrorHandler
  implements ErrorHandler {

  handleError(error: unknown) {
    console.error(error);
  }
}
Error Boundary
class ErrorBoundary extends Component {
  state = { hasError: false };

  static getDerivedStateFromError() {
    return { hasError: true };
  }
  componentDidCatch(err, info) {
    console.error(err, info);
  }
  render() {
    if (this.state.hasError)
      return <h1 className="text-red-500">
               Something went wrong.
             </h1>;
    return this.props.children;
  }
}

Route Guards

CanActivate Guard
@Injectable({ providedIn: 'root' })
export class AuthGuard
  implements CanActivate {

  canActivate(): boolean {
    return !!localStorage.getItem('token');
  }
}
// Route config:
{ path: 'dashboard',
  component: DashboardComponent,
  canActivate: [AuthGuard] }
ProtectedRoute component
function ProtectedRoute({ children }) {
  const isLoggedIn =
    !!localStorage.getItem('token');
  return isLoggedIn
    ? <>{children}</>
    : <Navigate to="/login" replace />;
}

// Usage:
<Route path="/dashboard"
  element={
    <ProtectedRoute>
      <Dashboard />
    </ProtectedRoute>
  } />
