01

Pipeline Structure

Jenkins
🧱
Two Pipeline Styles We use Scripted style for Pipeline 1 (node { }) and Declarative style for Pipeline 2 (pipeline { }). Both are valid. Scripted is more flexible. Declarative has more structure and built-in features like post, parameters, environment.
● Scripted Pipeline (Pipeline 1)
// Variables declared here live for all stages
node {
    def myVar = ''

    stage('Checkout') {
        // runs git pull etc
    }

    stage('Build') {
        // runs dotnet build etc
    }

    stage('Trigger') {
        // fires Pipeline 2
    }
}
● Declarative Pipeline (Pipeline 2)
pipeline {
    agent any

    parameters {
        string(name: 'MY_PARAM', ...)
    }

    environment {
        MY_VAR = "value"
    }

    stages {
        stage('Stage Name') {
            steps {
                script {
                    // Groovy logic here
                }
            }
        }
    }

    post {
        success { echo "done" }
        failure { echo "failed" }
        aborted { echo "aborted" }
    }
}
⚠️
Helper functions must go OUTSIDE pipeline{ } In Declarative pipelines, def myFunction() { } must be declared before the pipeline { } block. Functions defined inside pipeline fail at runtime.
💡
script{ } block unlocks Groovy inside Declarative Inside steps { } you can only use Jenkins built-in steps. Wrap any Groovy logic (loops, if/else, variables) in a script { } block.

Groovy Concepts

Groovy
Variables Groovy

All variables use def. Declare at node{ } level to share across stages.

def name    = 'hello'       // String
def count   = 0            // Integer
def flag    = true         // Boolean
def list    = []           // Empty list
def map     = [:]          // Empty map
def result  = ''           // Empty string

// env.VAR = pipeline-level, survives across stages
env.SERVICE_TAGS = ''
env.DEPLOY_LOG   = ''
String Operations Groovy

String interpolation, splitting, trimming, contains checks.

def s = "CADB,SQLData,Workflow"

// Split into list
s.split(',')
// → ["CADB", "SQLData", "Workflow"]

// Interpolation — inject variable
"Hello ${s}"

// Trim whitespace
s.trim()

// Check if empty
!s?.trim()   // ? = null-safe

// Split by dot, get last part
"FinIQ.Services.CADB".tokenize('.').last()
// → "CADB"

// Substring from first {
raw.substring(raw.indexOf('{'))
Lists and Loops Groovy

.each{} loops, .collect{} transforms, .findAll{} filters.

def items = ["CADB", "SQLData", ""]

// Loop — return = skip to next (like continue)
items.each { item ->
    if (!item) return
    echo "${item}"
}

// Filter — remove empty strings
items.findAll { it }
// → ["CADB", "SQLData"]

// Transform — add prefix to each
items.collect { "ms-${it}" }
// → ["ms-CADB", "ms-SQLData", "ms-"]

// Join list into string
["a", "b", "c"].join(",")
// → "a,b,c"

// it = current item (automatic variable)
// same as $_ in PowerShell
Sorting + Max/Min Groovy

CPS-safe sorting using .collect{} + sort key. Used for version tag resolution.

def tags = ["1.0.0", "10.0.0", "1.0.9"]

// Convert each tag to sortable map
def latest = tags.collect { tag ->
    def p = tag.tokenize('.')
              .collect { it.toInteger() }
    [version: tag,
     sortKey: p[0] * 10000 + p[1] * 100 + p[2]]
}.max { it.sortKey }.version

// → "10.0.0"

// WHY: string sort gives wrong order
// "10.0.0" < "9.0.0" alphabetically
// integer sort: 100000 > 90000 ✅

// Get second-to-last (previous tag)
sorted[-2].version
// -1 = last, -2 = second to last
Try / Catch Groovy

Handle failures gracefully so one service doesn't block others.

try {
    // code that might fail
    bat "docker push ${image}"

} catch(Exception e) {
    // runs if try block throws
    echo "Failed: ${e.message}"

} finally {
    // ALWAYS runs — success or fail
    // used for cleanup (delete temp folders)
    bat "rmdir /S /Q temp"
}

// Stop pipeline immediately
error("Something went wrong")

// Mark as aborted (yellow) not failed (red)
currentBuild.result = 'ABORTED'
error("Nothing to do")
JSON Parsing Groovy

Parse JSON responses from skopeo, APIs, curl. Used for tag resolution and AI responses.

// Parse JSON string into Groovy map
def parsed = new groovy.json.JsonSlurper()
                .parseText(jsonString)

// Access fields
parsed.Tags                 // array
parsed.choices[0]           // first item
parsed.choices[0].message.content

// bat() output includes Windows prompt line
// strip it before parsing:
def raw  = bat(..., returnStdout: true).trim()
def json = raw.substring(raw.indexOf('{'))
def obj  = new groovy.json.JsonSlurper()
                .parseText(json)
Spaceship Operator <=> Groovy

Three-way comparison. Returns -1, 0, or 1. Used inside sort comparators.

1 <=> 2 // → -1 (1 is less than 2) 2 <=> 2 // → 0 (equal) 3 <=> 2 // → 1 (3 greater than 2) // Used in version comparison: def majorCmp = aParts[0] <=> bParts[0] if (majorCmp != 0) return majorCmp // if major differs → decided // if major equal → check minor next // Note: .sort{} with closure causes // CPS error in Jenkins pipeline. // Use .collect{}.max{} instead ✅
Triggering Another Job Groovy

Only Groovy can trigger other Jenkins jobs. PowerShell cannot.

// Fire and forget — Pipeline 1 closes
build job: "Pipeline2-JobName",
    parameters: [
        string(name: 'CHANGED_SERVICES',
               value: changedServices),
        string(name: 'MAJOR_MINOR',
               value: params.MAJOR_MINOR ?: '')
    ],
    wait: false   // don't block

// Wait for result (e.g. Selenium)
def result = build job: "SeleniumJob",
    wait: true,
    propagate: false  // don't auto-fail

if (result.result != 'SUCCESS') {
    // handle failure
}
03

PowerShell Commands

PowerShell
File Copy — Robocopy PowerShell

Robust file copy with exclusions. Used to copy published output excluding config files.

# Basic robocopy cmd /c "robocopy `"$src`" `"$dest`" /E /PURGE" # With exclusions and retry cmd /c "robocopy `"$src`" `"$dest`" /E /PURGE /R:3 /W:5 /XF web.config /XF file.xml" # Flags: # /E = copy all subfolders including empty # /PURGE = delete files in dest not in source # /XF = exclude file (one per file) # /XD = exclude directory # /R:3 = retry 3 times on failure # /W:5 = wait 5 seconds between retries # /NFL = no file list (cleaner output) # Exit codes 0-7 = success, 8+ = error $rc = $LASTEXITCODE if ($rc -gt 7) { throw "Robocopy failed: exit $rc" } $global:LASTEXITCODE = 0 # reset # IMPORTANT: trim trailing backslash # "D:\path\" breaks quote escaping $src = $src.TrimEnd([char]92)
File Copy — simple copy PowerShell

For copying specific files (like nginx.conf, Dockerfile) into temp build folder.

# Copy single file — /Y = overwrite copy /Y "$src\nginx.conf" "$dest\nginx.conf" # PowerShell native copy Copy-Item -Path "$src\file.conf" ` -Destination "$dest\file.conf" ` -Force # Copy entire folder recursively Copy-Item -Path "$src\*" ` -Destination $dest ` -Recurse -Force # Copy multiple specific files foreach ($file in @("nginx.conf", "default.conf", "Dockerfile")) { Copy-Item -Path "$nginxPath\$file" ` -Destination "$tempFolder\$file" ` -Force }
Folder Operations PowerShell

Create, delete, check existence of folders. Used for temp build folders and output dirs.

# Check if folder exists if (Test-Path $path) { ... } if (!(Test-Path $path)) { ... } # Create folder New-Item -ItemType Directory -Path $path -Force # Create folder (cmd style) mkdir "$path" # Delete folder recursively Remove-Item -Path $path -Recurse -Force # Delete folder (cmd style — faster) rmdir /S /Q "$path" # Create fresh temp folder bat """ if exist "$temp" rmdir /S /Q "$temp" mkdir "$temp" """ # Join paths safely $full = Join-Path $base $sub # better than "$base\$sub" (avoids double \\)
Path Operations PowerShell

Convert relative to absolute paths, get parts of a path. Used for csproj OutputPath resolution.

# Convert relative to absolute $abs = [System.IO.Path]::GetFullPath( (Join-Path $csproj.DirectoryName $relPath) ) # resolves ..\ and .\ in path # Get filename from path $file.FullName # full path $file.DirectoryName # folder only $file.Name # filename.ext $file.BaseName # filename only # Split path string by backslash $parts = $path.Split('\') # Get last part (folder/file name) $parts[-1] # Trim trailing backslash ALWAYS before # passing to robocopy or cmd /c $path = $path.TrimEnd([char]92)
Finding Files PowerShell

Search for files by pattern. Used to find .csproj files inside service folders.

# Find first .csproj recursively $csproj = Get-ChildItem $path ` -Filter *.csproj ` -Recurse | Select-Object -First 1 # Find all .json files Get-ChildItem $path -Filter *.json -Recurse # Check result is not null if (!$csproj) { Write-Host "No csproj found" continue } # Read XML file (.csproj is XML) [xml]$xml = Get-Content $csproj.FullName # Navigate XML nodes $node = $xml.Project.PropertyGroup | Where-Object { $_.OutputPath } | Select-Object -Last 1
String Operations PowerShell

Split, join, trim, replace. Used for service name extraction and exclude flag building.

# Split string "a,b,c".Split(",") # → ["a","b","c"] "a.b.c".Split(".") # → ["a","b","c"] # Get last element $arr[-1] # last item $service.Split('.')[-1] # last part # Join array into string $arr -join "," # → "a,b,c" # Build exclude flags for robocopy $excludeStr = ($env:EXCLUDE_FILES.Split(",") | ForEach-Object { "/XF $($_.Trim())" }) -join " " # → "/XF web.config /XF file.xml" # $_ = current item in loop (like 'it' in Groovy) # $() = subexpression to evaluate inside string
Git Commands PowerShell

Detect changed files between commits. Core of Pipeline 1 change detection.

# Pull latest from remote git pull origin master # Get changed files vs previous commit $result = git diff --name-only HEAD~1 HEAD 2>$null # → list of changed file paths # e.g. Source_Code/8.0/API/FinIQ.Services.CADB/File.cs # 2>$null = suppress error output # Parse paths to extract service name foreach($name in $result) { $parts = $name.Split("/") $idx = $parts.IndexOf("API") if ($idx -ge 0) { $service = $parts[$idx + 1] } } # Add safe directory (run once) git config --global --add safe.directory $path
dotnet Commands PowerShell

Build and publish .NET projects. Core of the MS build stage.

# Restore NuGet packages dotnet restore $csproj.FullName | Out-Host # Publish (build + output) dotnet publish $csproj.FullName ` -c Release ` -o $outputPath ` --no-restore | Out-Host # -c Release = Release configuration # -o = output directory # --no-restore = skip restore (already done) # | Out-Host = send output to console log # NOT captured by returnStdout # Check if command succeeded if ($LASTEXITCODE -ne 0) { Write-Host "Build failed for $service" continue # skip to next service }

Groovy ↔ PowerShell Bridge

Both
🚨
The Golden Rule — Single vs Double Quotes Use '''single quotes''' when PowerShell variables like $service must stay as PowerShell variables. Use """double quotes""" when you need Groovy to inject a value first — but escape every PowerShell $ as \$ and every \ as \\\\.
Single Quotes — PowerShell variables stay as-is
powershell(script: '''
    # Groovy leaves everything alone
    # $service is PowerShell variable ✅
    $service = "CADB"
    Write-Host $service
    $path = "D:\some\path"
    $arr = $items -join ","
''')
✅
Use single quotes when the script is self-contained and doesn't need values from Groovy variables.
withEnv — cleanest way to pass Groovy → PowerShell
withEnv([
    "OUTPUT_PATH=${outputPath}",
    "EXCLUDE_FILES=${excludeFiles}"
]) {
    powershell(script: '''
        # Read Groovy values as env vars
        $dest    = $env:OUTPUT_PATH
        $exclude = $env:EXCLUDE_FILES
        # No escaping needed ✅
    ''')
}
✅
Use withEnv when you need to pass Groovy variables into PowerShell without mixing quote styles.
Capturing PowerShell output back into Groovy
// returnStdout: true captures EVERYTHING printed to stdout
// Write-Host → goes to Jenkins console log (NOT captured)
// Write-Output → goes to stdout (IS captured)
// | Out-Host → forces to console log (NOT captured)

def result = powershell(
    returnStdout: true,
    script: '''
        Write-Host "This goes to console only"      # not captured
        dotnet publish ... | Out-Host               # not captured
        cmd /c robocopy ... | Out-Host              # not captured
        $services = @("CADB","SQLData")
        Write-Output ($services -join ",")          # CAPTURED ✅
    '''
).trim()
bat() vs powershell() — when to use which
// bat() — runs Windows cmd commands
// use for: docker, helm, oc, skopeo, robocopy, git
bat "docker push ${fullImage}"
bat "helm upgrade --install ms-${service} ..."

// bat() with returnStdout — output includes Windows prompt line
// always strip it:
def raw  = bat(script: "skopeo list-tags ...", returnStdout: true).trim()
def json = raw.substring(raw.indexOf('{'))  // strip prompt line

// powershell() — runs PowerShell scripts
// use for: file ops, path resolution, git diff, dotnet, loops
powershell(script: '''
    # multi-line PowerShell logic
''')

// Multi-line bat with line continuation ^
bat """
    helm upgrade --install ms-${service} ${helmPath} ^
    -f ${helmPath}\\ms-values\\ms-${service}-values.yaml ^
    --set image.tag=${tag} ^
    -n ${namespace}
"""

Common Patterns

Patterns
Pattern 1 — Process Multiple Services with Skip on Failure Both 
// Standard pattern used in both MS and MFE pipelines
params.CHANGED_SERVICES.split(',')
    .findAll { it }          // remove empty strings
    .each { service ->

        service = service.trim()

        // Skip invalid entries
        if (!service || service.contains(' ')) {
            echo "Skipping invalid: [${service}]"
            return   // = continue in Groovy .each{}
        }

        try {
            // do work per service
            env.SERVICE_TAGS += "${service}=${image},"

        } catch(Exception e) {
            echo "Failed for ${service}: ${e.message}"
            return   // skip this service, continue with next
        }
    }
Pattern 2 — Credentials Block Groovy 
// Username + Password (Quay, Bitbucket, OCP)
withCredentials([usernamePassword(
    credentialsId: "my-creds-id",
    usernameVariable: 'MY_USER',
    passwordVariable: 'MY_PASS'
)]) {
    bat "docker login -u %MY_USER% -p %MY_PASS%"
    // %VAR% = Windows env var syntax in bat()
    // $env:VAR = PowerShell env var syntax
}

// Single secret string (API keys, tokens)
withCredentials([string(
    credentialsId: "my-api-key-id",
    variable: 'API_KEY'
)]) {
    bat "curl -H \"api-key: %API_KEY%\" ..."
}

// Credentials are MASKED in Jenkins logs automatically
// Never hardcode passwords in Jenkinsfile
Pattern 3 — Auto Rollback Flow Groovy
// Deploy → health check → helm rollback → previous tag // Each step only runs if previous failed bat "helm upgrade --install ms-${svc} ..." // Step 1 deploy if (isPodHealthy(svc, namespace)) { return } // ✅ done // Step 2 — helm rollback bat "helm rollback ms-${svc} -n ${namespace}" sleep(time: 30, unit: 'SECONDS') if (isPodHealthy(svc, namespace)) { return } // ✅ done // Step 3 — re-deploy previous Quay tag def prevTag = getPreviousTag(svc, ...) if (prevTag) { bat "helm upgrade --install ms-${svc} ... --set image.tag=${prevTag}" } else { echo "No previous tag — manual intervention needed" } // ALWAYS notify regardless of outcome sendFailureNotification(svc, tag, namespace, reason, rollbackResult)
Pattern 4 — Temp Folder Build Context (MFE) PowerShell 
// Create → populate → build → cleanup
// finally{} ensures cleanup even on failure

try {
    // 1. Create fresh temp folder
    bat """
        if exist "${temp}" rmdir /S /Q "${temp}"
        mkdir "${temp}"
    """

    // 2. Copy dist files
    bat """
        robocopy "${distSrc}" "${temp}\\dist" /E /R:3 /W:5
        if %ERRORLEVEL% GTR 7 exit /b 1
    """

    // 3. Copy nginx files
    bat """
        copy /Y "${nginxPath}\\nginx.conf"   "${temp}\\nginx.conf"
        copy /Y "${nginxPath}\\default.conf" "${temp}\\default.conf"
        copy /Y "${nginxPath}\\Dockerfile"   "${temp}\\Dockerfile"
    """

    // 4. Docker build
    bat "docker build -t ${image} \"${temp}\" --no-cache"
    bat "docker push ${image}"

} catch(Exception e) {
    echo "Failed: ${e.message}"

} finally {
    // ALWAYS delete temp — success or fail
    bat "if exist \"${temp}\" rmdir /S /Q \"${temp}\""
}

⚠️
Robocopy exit codes 0-7 = success, 8+ = error Always capture $rc = $LASTEXITCODE immediately after robocopy, then check if ($rc -gt 7). Then reset $global:LASTEXITCODE = 0 so Jenkins doesn't see non-zero exit.
⚠️
bat() output includes Windows prompt line When using returnStdout: true with bat(), always strip the prompt: raw.substring(raw.indexOf('{')) for JSON or .trim() for plain text.
🚨
Trailing backslash breaks robocopy quotes Path ending with \ inside quotes becomes "D:\path\" which escapes the closing quote. Always call .TrimEnd([char]92) on both source and destination paths before robocopy.
🚨
.sort{closure} causes CPS error in Jenkins Never use list.sort { a, b -> } in Declarative pipeline. Use .collect{ [version: it, sortKey: ...] }.max { it.sortKey }.version instead.
💡
return inside .each{} = continue, not return In Groovy, return inside .each{} skips to the next iteration — it does NOT exit the parent function. To exit the parent function, use a boolean flag or restructure the logic.
💡
Out-Host prevents stdout capture Pipe any command whose output you don't want captured (dotnet, robocopy) to | Out-Host. Only Write-Output at the very end should reach returnStdout.
✅
Helper functions go before pipeline{ } In Declarative pipelines, define all def functionName() { } blocks above the pipeline { } keyword. They cannot live inside stages or steps.
✅
finally{} for temp folder cleanup Always use try { } catch { } finally { } when creating temp folders. The finally block runs whether the build succeeded or failed — temp folder always gets deleted.
