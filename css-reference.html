display
The most important CSS property. It controls how a div behaves — does it stack, sit inline, form a grid, or disappear?
display: blockTakes full width of parent. Always starts on a new line. This is the DEFAULT for div, p, h1.
Tailwind:block
Think of it like: A paragraph in a book. Each paragraph always starts on a new line and goes full width across the page.

Think of it like: A word inside a sentence. It just sits in the flow of text — doesn't start a new line, doesn't take up extra space.

min-width: 0One of the most confusing CSS properties. By default flex children cannot shrink below their content width. min-width:0 removes this restriction.
Tailwind:min-w-0
Why this is confusing: You set flex:1 on a panel expecting it to share space. But it won't shrink below the width of its content — like a paragraph of very long text — even though you said flex:1. The browser says "I want to flex:1 BUT my content is 800px wide so I need at least 800px". min-width:0 says "forget about your content size, you can shrink as small as needed and the content will clip or wrap instead".
Rule: Whenever you have flex:1 on something that also contains text (especially long code lines), add min-width:0 to it. Without it, text overflow will break the layout.

box-sizing
box-sizing: border-boxChanges how width and height are calculated. border-box includes padding and border IN the width. content-box (default) does NOT.
Tailwind:Tailwind includes box-sizing:border-box by default in its reset
The problem without border-box:
You set width:200px and padding:10px.
content-box → actual rendered width = 200 + 10 + 10 = 220px (broke your layout!)
border-box → actual rendered width = 200px exactly (padding is INSIDE)
Always use this: Put *, *::before, *::after { box-sizing: border-box; } at the top of every CSS file. Your app already does this. You never need to think about this problem again.
calc()
calc()Mix different units in one value. Let the browser do the math.
Tailwind:h-[calc(100%-320px)]w-[calc(100vw-56px)]
/* In your app — Monaco height in judge mode: */ height={mode === "judge" ? "calc(100% - 320px)" : "100%"} /* Without calc, you can't mix units: */ /* height: 100% - 320px ← this DOES NOT WORK */ /* height: calc(100% - 320px) ← this WORKS */ /* Other examples: */ width: calc(50% - 8px); /* half width minus gap */ height: calc(100vh - 48px); /* full window minus topbar */
calc() is one of the most useful CSS features. Whenever you need "full width minus some pixels" or "half minus gap", use calc().

overflow
What happens when content is BIGGER than its container? You have four choices.
overflowControls what happens when content overflows the container boundaries.
Tailwind:overflow-visibleoverflow-hiddenoverflow-autooverflow-scrolloverflow-x-hiddenoverflow-y-auto
Think of a fish tank (container) and water (content):

overflow:visible → the water spills out over the sides. The tank looks broken.
overflow:hidden → an invisible lid on the tank. Water is capped at the tank edge.
overflow:auto → lid appears only when water level reaches the top. Otherwise open.
overflow:scroll → lid with a pump always visible even if the tank is not full.
Values
visibleoverflow-visibleContent spills outside. Default. Almost never what you want in your layouts.
hiddenoverflow-hiddenClips content at the container edge. Used on .tc-item to clip analysis box to rounded corners.
autooverflow-autoScrollbar appears ONLY when content overflows. Clean. Used on output scroll area and TC list.
scrolloverflow-scrollScrollbar always visible even when content fits. Usually ugly — prefer auto.
overflow-x / overflow-yoverflow-x-hidden overflow-y-autoControl horizontal and vertical separately.
This content is 500px wide — much wider than the container above
overflow:hidden has a bonus effect: When a parent has border-radius AND overflow:hidden, children get clipped to that rounded shape. This is why your TC cards use it — the analysis box background clips to the card's 8px border-radius.

Normal wrapping only breaks at spaces. But what if there are no spaces? A URL like "https://very-long-domain-name-that-goes-on-forever.com/path/to/something" has no spaces. It will overflow.
overflow-wrap:break-word says "if a word is too long, break it mid-word at the container edge". Used in your analysis box pre tag.

user-select
user-select: nonePrevents text from being selected when the user clicks and drags. Used on logos, labels, UI chrome.
Tailwind:select-noneselect-textselect-all
When to use it: Any text the user would never want to copy — logo name, language abbreviations (PY, JS), tab labels. Without it, clicking a button might accidentally select the button text.

pointer-events
pointer-events: noneMakes an element completely ignore all mouse interactions — clicks, hover, everything passes through it.
Tailwind:pointer-events-nonepointer-events-auto
Think of it like: A glass window. You can see through it but you can't interact with it — your click goes through to whatever is behind it.
Common use: Overlays and loading states. If you have a loading spinner on top of a button, add pointer-events:none to the spinner so clicks still reach the button behind it.

units cheatsheet
Every unit you will encounter in CSS, explained plainly.
all units
Absolute units
pxw-[10px]Pixels. Always the same size regardless of screen or parent. Most used in your app for borders (1px), padding (10px, 16px), fixed component sizes (56px sidebar, 44px topbar).
Relative units
%w-full h-fullPercentage of the PARENT's size. width:100% = same width as parent. height:100% = same height as parent. Requires parent to have a defined size.
vhh-screenViewport height. 1vh = 1% of browser window height. 100vh = full browser window. Used on your outermost app div.
vww-screenViewport width. 100vw = full browser width.
emRelative to the CURRENT element's font-size. If font-size is 14px: 1em=14px, 0.5em=7px. Problem: it compounds (nested elements stack up). Less predictable. Used in letter-spacing in your app.
remRelative to the ROOT element's font-size (usually 16px in browsers). Always consistent regardless of nesting. 1rem=16px always. Better than em for font sizes and spacing.
frgrid-cols-[1fr_1fr]Fraction. Grid ONLY. 1fr = 1 fraction of available space. 1fr 1fr = two equal columns. 2fr 1fr = first column twice as wide as second.
Special
calc()h-[calc(100%-320px)]Mix units in one value. Used for Monaco height in judge mode. calc(100% - 320px) = "full height minus exactly 320 pixels".
autow-auto h-autoBrowser calculates. For width = fit content. For height = fit content. For margin = distribute remaining space (centering trick: margin:0 auto).
min-contentShrink as small as possible without breaking content. Rarely used directly.
max-contentGrow as large as content needs. No wrapping.
